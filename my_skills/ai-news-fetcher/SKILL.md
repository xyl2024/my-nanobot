---name: ai-news-fetcherdescription: |  抓取并展示 AI 新闻资讯。使用场景：(1) 用户要求获取 AI 行业新闻/热点/资讯 (2) 用户让你搜集今日 AI 新闻 (3) 用户给你一个新闻网站让你抓取内容 (4) 用户询问今天 AI 有什么新闻。默认使用 https://ai.hubtoday.app/ 进行抓取，也可接受用户指定的其他 AI 新闻网址。---# AI News Fetcher从 AI 新闻网站抓取最新资讯并完整展示。## 核心原则（必须严格遵守）**零修改原则**：- ❌ 禁止添加任何标题、前言、摘要、总结- ❌ 禁止删除任何内容（除非明显重复）- ❌ 禁止改写任何文字- ❌ 禁止添加emoji表情（如 🔥、🎯、✅ 等）- ❌ 禁止添加颜文字或装饰性符号- ✅ **100% 保留原始内容，仅做必要的格式转换**## 清理规则（获取内容后必须执行）获取内容后，必须清理以下内容：1. ❌ 删除所有 emoji 表情符号2. ❌ 删除类似 `(AI资讯)`、`(✧∀✧)` 等后缀标注3. ❌ 删除 Markdown 锚点链接（如 `[](#xxx)`）4. ❌ 删除无意义的装饰符号，保留必要的加粗等格式5. ✅ 保留原文的标题、列表、链接、代码块等结构6. ✅ 保留 GitHub 仓库完整地址## 使用方法1. 使用 `web_fetch` 工具抓取网页内容：   - 默认网站：`https://ai.hubtoday.app/`   - 用户指定其他网址时，使用用户提供的 URL2. 获取内容后，按照清理规则处理3. 输出清理后的内容## 格式要求- 使用 Markdown 格式输出- 保留原文的标题、列表、链接、代码块等所有格式- **重要**：所有 GitHub 仓库地址必须完整保留，格式如 `https://github.com/owner/repo`- 在项目列表中，星标信息可以和 GitHub 地址一起展示（如 `owner/repo ⭐1234`）## GitHub 链接处理规则- 绝对不能删除或简化 GitHub 仓库地址- 即使做简化处理，也要保留完整的 GitHub 链接（例如：[owner/repo](https://github.com/owner/repo)）## 示例触发方式- "帮我搜集今日 AI 热点新闻"- "今天 AI 有什么新闻？"- "抓取这个网站的内容：xxx"- "帮我看看 AI 新闻"## 注意事项- 某些网站可能有反爬机制，抓取失败时告知用户- 如果是多页内容，询问用户是否需要继续抓取其他页面
